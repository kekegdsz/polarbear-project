# PolarBear 项目文档

仓库包含两个相互配合的子项目：**Spring Boot REST API** 负责用户、版本、订单等数据与鉴权；**Vue 3 站点** 提供 App 分发落地页、工具页与管理后台，通过浏览器请求同源或反向代理下的 `/api` 与后端通信。

---

## 1. 仓库结构

| 目录 | 角色 |
|------|------|
| `springboot-server-api/` | 后端 API（Java 17、Spring Boot、MyBatis） |
| `vue-web-server/` | 前端（Vue 3、Vite、Vue Router） |

---

## 2. 后端：`springboot-server-api`

### 2.1 技术栈

- Java 17、Spring Boot 4.0.x（打包为 **WAR**，可用 `java -jar` 直接运行）
- MyBatis、Spring Validation、Spring Security Crypto（密码哈希）
- 开发默认：**H2 内存库** + `schema.sql` 初始化
- 生产：**MySQL**（见 `application-prod.yml`）
- API 文档：**SpringDoc OpenAPI**，Swagger UI

### 2.2 配置要点

- `server.servlet.context-path` 为 **`/api`**，因此业务路径均以 `/api` 为前缀（例如 `/api/hello`）。
- 开发环境默认端口在 `application.yml` 中为 **8081**（与生产文档中常见的 8082 可并存，以实际部署参数为准）。
- 生产激活 profile：`--spring.profiles.active=prod`，数据源与 MySQL 账号以 `application-prod.yml` 为准；数据库密码建议通过环境变量 **`MYSQL_PASSWORD`** 注入，勿将真实密码写入公开文档或提交到仓库。

### 2.3 分层结构（摘自子项目 README）

```
src/main/java/com/undersky/api/springbootserverapi/
├── config/          # 配置（含管理端鉴权拦截器等）
├── controller/      # 控制器
├── service/         # 业务逻辑
├── repository/      # 数据访问（若使用）
├── mapper/          # MyBatis Mapper 接口
├── model/
│   ├── dto/         # 请求/统一封装
│   ├── entity/      # 实体
│   └── vo/          # 对外视图对象
├── exception/       # 全局异常
└── constant/        # 常量
```

### 2.4 统一响应格式

成功时典型形态：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

- `code === 0` 表示成功，非 0 为失败  
- 管理端接口需在请求头携带 **`X-Auth-Token`**（登录后返回的 token）

### 2.5 接口概览（代码与现有 API 文档对照）

以下路径均相对于 **`/api`**（即完整 URL 形如 `http://主机:端口/api/...`）。

| 分类 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 示例 | GET | `/hello` | Hello World |
| 认证 | POST | `/auth/register` | 注册（设备 UUID 或用户名密码） |
| 认证 | POST | `/auth/login` | 登录 |
| 公开版本 | GET | `/versions/latest` | 最新已发布版本，`channel` 可选 `android` / `ios` / `ohos` |
| 管理端 | GET | `/admin/profile` | 当前管理员信息 |
| 管理端 | PUT | `/admin/profile/password` | 修改密码 |
| 管理端 | GET | `/admin/users/stats` | 用户统计 |
| 管理端 | GET | `/admin/users` | 用户列表（分页等以 Swagger 为准） |
| 管理端 | GET 等 | `/admin/versions` | 版本列表/发布/删除（见 Swagger） |
| 管理端 | GET 等 | `/admin/orders` | 订单统计与列表（见 Swagger） |

更完整的请求体、字段说明与示例见子项目 **[springboot-server-api/docs/API.md](../springboot-server-api/docs/API.md)**。  
在线文档：部署后访问 **`/api/swagger-ui.html`**，OpenAPI JSON：**`/api/v3/api-docs`**。

### 2.6 本地运行与打包

```bash
cd springboot-server-api
./mvnw spring-boot:run
```

默认可访问：`http://localhost:8081/api/hello`（端口以 `application.yml` 为准）。

```bash
./mvnw clean package -DskipTests
```

产物：`target/springboot-server-api-0.0.1-SNAPSHOT.war`。

## 3. 前端：`vue-web-server`

### 3.1 定位与技术栈

- **Vue 3** + **Vite 5** + **Vue Router 4**（History 模式）
- 站点主题：App 分发介绍、下载入口、深色风格与响应式布局
- 额外页面：API 说明、JSON 工具、Icon 制作工具等（见路由配置）

### 3.2 主要功能与路由

| 路径 | 说明 |
|------|------|
| `/` | 首页（应用展示与下载） |
| `/api-doc` | API 接入说明页 |
| `/json-tool` | JSON 格式化工具 |
| `/icon-maker` | App Icon 工具 |
| `/admin/login` | 管理后台登录 |
| `/admin/users` | 用户管理 |
| `/admin/versions` | 版本管理 |
| `/admin/orders` | 订单管理 |

管理路由通过 `localStorage` 中的 `admin_token` 与 `admin_role === 'admin'` 做前端守卫；真实权限以后端校验为准。

### 3.3 业务配置

应用名称、描述、版本号与各平台下载链接在 **`vue-web-server/src/config/app.js`** 的 `appConfig` 中维护，发布前按需修改。

### 3.4 开发时的 API 代理

**`vite.config.js`** 中将 `/api` 代理到后端地址（当前仓库中示例指向远程主机，本地联调时请改为本机后端，例如 `http://127.0.0.1:8081`），以便 `fetch('/api/...')` 在开发环境可达。

### 3.5 本地命令

```bash
cd vue-web-server
npm install
npm run dev      # 开发（Vite 默认与配置中的 server.port 一致，当前为 8082）
npm run build    # 产出 dist/
npm run preview  # 本地预览构建结果
```

---

## 4. 前后端协作关系

1. 浏览器访问前端静态资源（Nginx 或 CDN）。  
2. 前端通过相对路径 **`/api/...`** 调用后端；生产环境通常由 **Nginx 将 `/api` 反代** 到内网 Spring Boot 进程监听的端口。  
3. 公开客户端可通过 **`GET /api/versions/latest`** 获取各渠道最新版本信息；管理端在登录后使用 `X-Auth-Token` 调用 `/api/admin/**`。
