# PolarBear 部署文档

本文说明如何将 **springboot-server-api** 与 **vue-web-server** 部署到生产环境（以常见 Linux 服务器 + 宝塔面板为例），并与子项目内已有文档保持一致。请将文中 **你的域名**、**服务器 IP**、**路径** 替换为实际值。

---

## 1. 部署架构建议

```
用户浏览器
    → 80/443 Nginx（静态：Vue dist + try_files）
    → location /api → 反向代理 → Spring Boot（如 127.0.0.1:8082）
```

- 前端：**纯静态**，无需 PHP/Java 运行时在 Nginx 站点进程内执行。  
- 后端：独立 JVM 进程，仅对 Nginx 或内网开放。

---

## 2. 后端部署：`springboot-server-api`

### 2.1 环境要求

- **JDK 17**  
- **MySQL**（生产 profile），数据库与账号与 **`springboot-server-api/src/main/resources/application-prod.yml`** 一致；密码务必用环境变量 **`MYSQL_PASSWORD`** 等方式注入，不要写进公开文档或代码仓库。

### 2.2 打包

```bash
cd springboot-server-api
./mvnw clean package -DskipTests
```

生成：`target/springboot-server-api-0.0.1-SNAPSHOT.war`（可用 `java -jar` 运行）。

### 2.3 启动参数示例

- 指定端口（若 8081 被占用可改用 8082 等）：

```bash
java -jar springboot-server-api-0.0.1-SNAPSHOT.war --server.port=8082
```

- 激活生产配置：

```bash
java -jar springboot-server-api-0.0.1-SNAPSHOT.war \
  --server.port=8082 \
  --spring.profiles.active=prod
```

- 建议同时设置：`MYSQL_PASSWORD`（与数据库实际密码一致）。

后台运行可使用 `nohup ... &` 或 **systemd**（示例见子项目 **[宝塔部署指南.md](../springboot-server-api/docs/宝塔部署指南.md)**）。

### 2.4 宝塔「Java 项目管理器」要点

1. 运行方式：**JAR/WAR 运行**  
2. 启动文件：上述 WAR  
3. 运行参数示例：`--server.port=8082 --spring.profiles.active=prod`  
4. Java 路径指向 JDK 17 的 `java`

更细步骤见：**[springboot-server-api/docs/宝塔部署指南.md](../springboot-server-api/docs/宝塔部署指南.md)**。

### 2.5 MySQL 初始化

在服务器上创建库、用户并授权后，保证 `application-prod.yml` 中的 JDBC URL、用户名与之一致。具体操作可参考 **[springboot-server-api/docs/MySQL配置说明.md](../springboot-server-api/docs/MySQL配置说明.md)**（注意其中若含有示例密码，生产环境应改为强密码并仅保存在安全配置中）。

### 2.6 验证

- 健康检查：`http://服务器IP:端口/api/hello`  
- Swagger UI：`http://服务器IP:端口/api/swagger-ui.html`

### 2.7 一键上传脚本（可选）

子项目提供 **`springboot-server-api/scripts/deploy.sh`**，可将打包产物上传到服务器目录（如 `/opt/apps/springboot-server-api/`），具体以脚本内容为准；上传后仍需在服务器上配置启动命令或宝塔项目。

## 3. 前端部署：`vue-web-server`

### 3.1 构建

```bash
cd vue-web-server
npm install
npm run build
```

产物在 **`dist/`**，将 **dist 目录内的所有文件** 上传到网站根目录（不是上传 dist 文件夹本身）。

### 3.2 Nginx（History 模式必配）

Vue Router 使用 **history** 模式，必须在 `server { }` 内为静态站点配置：

```nginx
location / {
    root   /www/wwwroot/你的站点目录;
    index  index.html;
    try_files $uri $uri/ /index.html;
}
```

保存后重载 Nginx。完整说明见 **[vue-web-server/DEPLOY.md](../vue-web-server/DEPLOY.md)**。

### 3.3 与后端同域

将 API 反代到本机后端，避免浏览器跨域（路径需与后端 `context-path` 一致，即 **`/api`**）：

```nginx
location /api {
    proxy_pass http://127.0.0.1:8082/api;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 60s;
}
```

若后端端口不是 8082，请修改 `proxy_pass` 中的端口。示例亦见后端 **[宝塔部署指南.md](../springboot-server-api/docs/宝塔部署指南.md)**。

### 3.4 参考配置片段

仓库内另有示例：`vue-web-server/deploy/vue-spa.conf`、`vue-web-server/deploy/tianyi-web.conf`、`vue-web-server/nginx.conf.example`，可按环境裁剪使用。

### 3.5 前端部署脚本（可选）

**`vue-web-server/deploy.sh`** 可用于自动化构建与上传，使用前请阅读脚本内变量与目标路径。

## 4. 网络与安全组

- 对外通常只开放 **80 / 443**；后端端口（如 8082）建议仅 **127.0.0.1** 访问或通过防火墙限制来源。  
- 若网站使用 **80** 端口，需在云厂商 **安全组** 中放行 TCP 80（及 443）。说明性步骤见 **[vue-web-server/doc/开放80端口.md](../vue-web-server/doc/开放80端口.md)**（文中示例 IP 请忽略，以你的实例为准）。

## 5. 常见问题排查

### 5.1 调用 `/api/*` 返回 502

多为 **后端未启动**、崩溃或 Nginx **无法连上** 上游端口。

| 检查项 | 处理方向 |
|--------|----------|
| 进程是否存在 | `ps aux | grep springboot` 或 systemd 状态 |
| 本机直连 API | `curl http://127.0.0.1:8082/api/hello`（端口按实际） |
| Nginx 超时 | `location /api` 中增加 `proxy_read_timeout` 等 |
| 稳定运行 | 使用 systemd 并设置 `Restart=on-failure` |

更细的运维命令见 **[宝塔部署指南.md](../springboot-server-api/docs/宝塔部署指南.md)** 第七节。  
前端仓库中的 **[502排查说明.md](../vue-web-server/doc/502排查说明.md)**、**[502排查步骤.md](../vue-web-server/doc/502排查步骤.md)** 提供了按现象逐项排查的思路（含静态站点、CDN、多端口对比等），可将其中具体 IP/域名替换为你的环境。

### 5.2 前端路由 404

未配置 `try_files` 会导致刷新子路径 404。请确认 **「五、Nginx」** 一小节中的 `try_files`。

### 5.3 静态站点本身 502

若首页即 502 而非仅 API，可能是站点被错误地关联到 PHP/Java 运行环境。宝塔中可将站点改为 **纯静态**，同时保留 `/api` 的 `proxy_pass` 到后端。详见 **[502排查说明.md](../vue-web-server/doc/502排查说明.md)**。

### 5.4 开发环境代理错误

修改 **`vue-web-server/vite.config.js`** 中 `server.proxy['/api'].target`，指向你本机或测试环境后端，避免指向错误的远程地址。

## 6. 生产环境建议（摘要）

1. 使用 **`spring.profiles.active=prod`** 与独立 MySQL。  
2. JVM 可按机器规格设置 **-Xms / -Xmx**（子项目文档示例：`-Xms256m -Xmx512m`）。  
3. 配置 HTTPS 时在宝塔申请证书并监听 443。  
4. 日志：为 Spring Boot 配置持久化日志（如 logback 文件输出），便于排错。
