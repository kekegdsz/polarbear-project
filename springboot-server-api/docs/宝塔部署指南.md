# Spring Boot 项目 - 宝塔面板部署指南

## 一、前置准备

### 1. 宝塔面板需安装

- **Java 项目管理器**（软件商店搜索「Java」）
- **Nginx**（可选，用于反向代理）

### 2. 服务器需安装

- **JDK 17**（项目使用 Java 17）
- 宝塔安装 Java 后路径通常为：`/www/server/java/jdk-17.x.x/bin/java`

---

## 二、打包项目

在项目根目录执行：

```bash
./mvnw clean package -DskipTests
```

生成的包位置：`target/springboot-server-api-0.0.1-SNAPSHOT.war`

> Spring Boot 的 WAR 包可直接用 `java -jar` 运行，无需外部 Tomcat。

---

## 三、部署方式

### 方式 A：一键部署脚本（推荐）

```bash
./scripts/deploy.sh root@你的服务器IP
```

脚本会完成：打包 → 上传到 `/opt/apps/springboot-server-api/`

上传后在服务器执行启动命令（见脚本输出），或使用宝塔 Java 项目管理器。

### 方式 B：宝塔 Java 项目管理器

1. 打开宝塔面板 → **软件商店** → 安装 **Java 项目管理器**
2. 进入 **Java 项目** → **添加项目**
3. 填写：
   - **项目名称**：springboot-server-api
   - **项目路径**：如 `/opt/apps/springboot-server-api/`
   - **运行方式**：选择 **JAR/WAR 运行**
   - **启动文件**：`springboot-server-api-0.0.1-SNAPSHOT.war`
   - **运行参数**：`--server.port=8082`
   - **Java 路径**：`/www/server/java/jdk-17.0.8/bin/java`（按实际版本调整）
4. 点击 **启动**

> 若 8081 被 Nginx 等占用，使用 `--server.port=8082`

### 方式 C：命令行启动

```bash
cd /opt/apps/springboot-server-api
/www/server/java/jdk-17.0.8/bin/java -jar springboot-server-api-0.0.1-SNAPSHOT.war --server.port=8082
```

后台运行：

```bash
nohup /www/server/java/jdk-17.0.8/bin/java -jar springboot-server-api-0.0.1-SNAPSHOT.war --server.port=8082 > app.log 2>&1 &
```

### 方式 D：Systemd 服务

创建 `/etc/systemd/system/springboot-api.service`：

```ini
[Unit]
Description=Spring Boot API Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/apps/springboot-server-api
ExecStart=/www/server/java/jdk-17.0.8/bin/java -jar /opt/apps/springboot-server-api/springboot-server-api-0.0.1-SNAPSHOT.war --server.port=8082
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable springboot-api
sudo systemctl start springboot-api
```

## 四、验证部署

- API：`http://你的服务器IP:8082/api/hello`
- 接口文档（Swagger UI）：`http://你的服务器IP:8082/api/swagger-ui.html`

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": "Hello World"
```

## 六、Nginx 反向代理（可选）

若需通过 80 端口或域名访问，在宝塔添加反向代理：

- **目标 URL**：`http://127.0.0.1:8082`
- **代理目录**：`/api`

配置示例：

```nginx
location /api {
    proxy_pass http://127.0.0.1:8082/api;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

## 七、502 错误排查

登录等接口返回 502 时，常见原因：

| 原因 | 处理 |
|------|------|
| 后端未运行 | 执行 `./start-prod.sh` 或启用 systemd 服务 |
| Nginx 连接超时 | 在 `location /api/` 中增加 `proxy_read_timeout 60s;` 等 |
| 域名未配置 | 在 `server_name` 中加入你的域名（如 undersky.tech） |

**推荐**：使用 systemd 实现崩溃自动重启，减少 502：

```bash
sudo cp /tmp/springboot-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable springboot-api
sudo systemctl start springboot-api
```

## 八、生产环境建议

1. **环境切换**：`--spring.profiles.active=prod`
2. **JVM 参数**：`-Xms256m -Xmx512m`
3. **防火墙**：放行 8082 端口
4. **日志**：建议配置 logback 输出到文件
