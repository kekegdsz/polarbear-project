# 502 错误排查说明

## 一、502 多半是 Java 后端未启动（最常见）

访问首页正常，但登录、管理页等调用 `/api/*` 时 502，说明 **Java 后端 (8082) 未运行**。

**一键修复（推荐）：**

```bash
bash deploy/fix-502.sh
```

或仅重启后端：

```bash
bash deploy/restart-backend.sh
```

或手动 SSH 执行：

```bash
ssh root@183.56.251.215 "systemctl restart springboot-api"
```

**官网地址：** http://undersky.tech/ （80 端口）或 http://undersky.tech:8081/  
**注意：** 请使用 http:// 访问，不要用 https://（未配置 SSL）

---

## 二、先试备用端口 8080

已在服务器添加 **8080** 端口站点，请访问：

**http://183.56.251.215:8080/**

- 若 8080 能访问 → 多半是 80 端口被拦截，需在云控制台安全组放行 8080
- 若 8080 也 502 → 多半是站点运行环境配置问题

---

## 三、在宝塔里把站点改为「纯静态」（仅静态页 502 时）

若首页 `/` 就 502（非 API 调用），可能是 PHP 环境问题。此时可改为纯静态。注意：`/api/*` 仍由 Nginx 代理到 Java，需确保 Java 后端已启动。

**操作步骤：**

1. 登录宝塔面板 → **网站**
2. 找到 `183.56.251.215` 站点 → 点击 **设置**
3. 左侧点击 **PHP 版本**（或「运行环境」）
4. 选择 **纯静态**
5. 保存

纯静态后，Nginx 直接返回静态文件，不再转发到 PHP/Java。

---

## 四、确认云厂商安全组放行端口

在云控制台（阿里云 / 腾讯云 / 华为云等）中为安全组添加入站规则：

| 端口 | 协议 | 来源     |
|------|------|----------|
| 80   | TCP  | 0.0.0.0/0 |
| 8080 | TCP  | 0.0.0.0/0 |

8080 为备用访问端口。

---

## 五、若仍 502，可尝试新建站点

1. 宝塔 → **网站** → **添加站点**
2. 域名：`183.56.251.215`（或新建一个域名）
3. 根目录：`/www/wwwroot/183.56.251.215`
4. **PHP 版本**：选 **纯静态**
5. 提交

然后把原来的站点删除或停用，避免冲突。
