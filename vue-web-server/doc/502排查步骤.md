# 502 排查步骤（能看到访问记录但打不开）

## 一、先做这几项测试

请依次访问，并记录**能打开**和**打不开**的地址：

| 测试 | 地址 | 结果 |
|-----|------|------|
| 1 | http://183.56.251.215/ |  |
| 2 | http://183.56.251.215/test.txt |  |
| 3 | http://183.56.251.215:8080/ |  |
| 4 | http://183.56.251.215:8080/test.txt |  |
| 5 | https://183.56.251.215/ |  |

- 若 **test.txt 能打开**、首页不能 → 可能是首页或前端资源有问题  
- 若 **8080 能打开**、80 不能 → 很可能是 80 端口被拦截（安全组/防火墙）  
- 若 **https 报错**、http 正常 → 需要配置 SSL  

---

## 二、是否在云厂商配置了 CDN / 负载均衡

如果用了：
- 阿里云 CDN
- 腾讯云 CDN
- 负载均衡 CLB / SLB

502 可能来自 CDN/负载均衡，而不是服务器上的 Nginx。

**处理：** 到对应云产品的控制台，查看：
1. 回源是否正常  
2. 源站地址和端口是否正确  
3. 回源超时、健康检查等配置  

---

## 三、换网络再试

- 手机用流量（4G/5G）访问  
- 换一个 WiFi 网络  
- 或换个城市/网络环境  

若换网络后能正常访问，多半是本地网络或运营商问题。

---

## 四、抓包 / 错误信息

1. 打开开发者工具（F12）→ **Network** 面板  
2. 再次访问并触发 502  
3. 点开失败的请求，查看：
   - **Status**（例如 502）  
   - **Response Headers**（是否有 `Server`、`X-` 等字段）  
4. 截图发给我，便于判断 502 是 Nginx 还是 CDN/负载均衡返回的  

---

## 五、服务器错误日志（需要你访问一次后再查）

你访问一次后，在服务器上执行：

```bash
ssh root@183.56.251.215 "tail -20 /www/wwwlogs/nginx_error.log"
```

或登录宝塔 → 网站 → 对应站点 → 日志，查看是否有 502 相关错误。
